# Adticks UI/UX Benchmark

## Reference Set Researched

- Ahrefs: strongest cue is evidence depth in Site Audit, especially crawl health, issue charts, and raw HTML/rendered HTML/page text inspection.
- Semrush: strongest cue is broad search intelligence plus explicit AI visibility workflows for brand presence, AI crawler barriers, competitor benchmarking, and AI answer surfaces.
- Ubersuggest: strongest cue is low-friction domain-first entry and accessible SEO language for smaller teams.
- Moz Pro: strongest cue is familiar all-in-one SEO suite framing around rankings, site crawl, optimization, and authority.
- Screaming Frog: strongest cue is technical crawler credibility, advanced extraction, JavaScript rendering, integrations, and dense table-first workflows.
- Sitebulb: strongest cue is audit education, prioritized hints, response-vs-render diagnostics, crawl maps, and visual explanations.
- Botify: strongest cue is enterprise SEO scale, log analysis, crawl-to-conversion thinking, and large-site operational confidence.

## UX Decisions Applied

- Reposition Adticks as a measurement platform, not a single-purpose audit tool.
- Keep the H1 as the brand name, but make the subheadline about measurable outcomes: discovery, AI citation, content quality, and technical health.
- Use an original measurement-console visual as the first-viewport scene instead of copying competitor dashboard screenshots.
- Add a domain-first input for familiarity, but frame the action as measuring a site rather than running a generic audit.
- Use a dark end-to-end interface because the brand identity is black/blue and the product should feel like an operations console.
- Add green, amber, and red signal accents so the page does not become a one-note blue interface.
- Make OAuth the primary conversion because real measurement data must be account-bound.
- Keep infrastructure details out of the landing-page story; implementation stack belongs in docs or enterprise technical due diligence, not the main conversion path.

## Design Principles

- Measurement beats checklist volume.
- Browser rendering must be positioned as a measurable signal, not the whole product.
- AI visibility should be a first-class measurement layer beside search, content, and technical health.
- Queue and worker state should be visible early because crawl economics will become a product differentiator.
- Copy must be original and should not reuse competitor slogans, section ordering, screenshots, or named feature labels.
